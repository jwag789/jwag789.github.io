In this blog post, I will be explaining classes in Ruby, and how and why to use them.

Classes

What are Ruby Classes? Ruby is an Object Oriented Programming Language, and involves objects and classes. A class is essentially a type of object, such as Array, String, or a class you create, and objects are instances of that class. Instances of that class will follow those classes rules. For example, your array "[1,5,3,6]" is an object, and it's class is Array. The methods defined in the Array class will only work for objects that were created from the Array class's initialize method. Let's do an example with our own class, which we will base on a real life object.

Creating an "Album" class

This image shows the different steps it takes to define a class, and to define an instance of that class. It also shows how to call instance methods on those objects. We start by opening the Album class, and define its initialize method. Here, we pass in arguments that will become instance variables, or variables that we can use on any of the methods in the class, that will work on an instance of that class. You can also make instance variables that isn't also an argument for the method, such as my "signed" variable. It will always start off as false, so I don't need to pass it through initialize, where the other variables will always be different. Instance variables are helpful, because now I can use them in any method. For example, I'm "@year" to define my outcome when I "play" my album, and the outcome of my "@value" of a signed album, which I also use in the value method. I wanted to make instance methods (which are methods that will only work on instances of that class, rather than on the class itself) like "play" and something to get the album signed, to compare it to a real life object. Think of the variables we saved as your instance or your copy of that album. It may or may not be signed. That all depends on what you do with your album. If you hand it off to a rocker (use the method), you will get it signed. If the year is old enough, it will be worth much more.
