Enumerable#cycle

I found cycle to be an interesting method to use. When using cycle, you call it after an object, and give it a block to pass. The method will then cycle through each element in the array, and perform the action that is in the block. If an argument is given after the cycle, it will cycle through that amount of times. If no argument is given, it will cycle infinitely. Changes made through the first cycle will have no affect on the next cycles, and it will always be the same.

For example, look at the code pictured in this post. I take an array and call cycle on it. I used an argument of 3, so it will cycle through the array 3 times. I then use the block to puts the value of each number times 2. This will output the result 3 times, without changing it each time.

Cycle can be a really useful method if you know when to use it. If you just want to cycle through an array a certain amount of times, use this, but if you want to be able to be more specific and specify which elements in an array you want to cycle through, use something like each. Also remember, if you use cycle to make an infinite loop, keep all necessary code inside the cycle block, otherwise it won't run!
