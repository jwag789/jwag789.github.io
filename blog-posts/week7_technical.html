This blog post is about the differences between SQL and NoSQL.

SQL vs NoSQL

SQL and NoSQL Databases have various features that make them different. First of all, when comparing SQL and NoSQL, I am talking about Relational Databases (RDBMS, for SQL) and non-relational, or distributed database (NoSQL). The NoSQL database structure differs from RDBMS and so some operations are faster in certain databases. Another main difference is that SQL is table based, where NoSQL databases are graph databases, wide column stores, document based, or key-value pairs. Therefore, SQL is simply tables with a certain amount of rows and columns, where NoSQL does not have those limitations. This goes hand in hand with the fact that SQL databases have predefined schema, where NoSQL does not.

A big difference between the two databases is syntax. SQL databases use, of course, SQL, or structured query language. NoSQL varies from database to database, and the queries are focused on a collection of documents, or the UnQL (Unstructured Query Language). SQL is better to perform complex queries, as NoSQL doesn't have a standard interface to do so, and the queries aren't as powererful as in SQL. SQL is also a good fit for large transactional type applications, as it's more stable. However, for heirarchical data, NoSQL is better, as it allows key-value pairing.

As far as scalability goes, there are differences there as well. SQL is typically vertically scalable, and you can increase the load by increasing the CPU/RAM/SSD/etc. on a server. NoSQL are horizontally scalable and you can add a few more servers easily in your database infrastructure.

All in all there are different advantages to using SQL or NoSQL. SQL is table based and is more stable with a powerful language. NoSQL has less limitations, uses documents, and is good for using key-value pairs and graph databases. I've dipped my toes into SQL and loved when I was able to figure how to use SELECT statements, and I'm excited to look into different NoSQL databases.
