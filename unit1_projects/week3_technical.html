This blog post is about why JavaScript is popular in web development, and the ups and downs of it.

JavaScript in Web Development

JavaScript is an easy to use, lightweight language, so it's no wonder it's become so popular to use in web development. When you hear about front end development, the languages that are ususally talked about are HTML, CSS, and JavaScript. They all go hand in hand, for the most part. Also it works on the client side rather than the server side, so it's fast. With Node.js there is so much that you can do, easily building "fast and scalable" network applications. This, along with browser consoles make testing very easy. There is also a large open source community.

There are downsides to the language, especially if you are big company that will be storing tons of data. It cannot access databases by itself. Also, the language doesn't distinguish the difference between floating point numbers and integers, and so using a language that is more exact might be a better fit for certain scenarios. It can't access web pages hosted on another domain, and it can't close a window that it did not open. The code can also not access anything outside of the web page.

Personally, while practicing Ruby, Python, and JavaScript, I found JavaScript to be the least readable. I see how it is helpful and easy to use with HTML, but the other languages had more "syntaxic sugar" and flowed better, in my opinion. Nevertheless, I also see its many uses, how its easy to use and test, and why it is so popular in web development.
